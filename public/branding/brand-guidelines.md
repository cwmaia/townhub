**Version:** 1.0
**Date:** November 25, 2025

## Brand Overview

**TownApp** is a mobile-first platform connecting residents and visitors with their local town's events, places, services, and real-time information. The brand identity reflects Nordic simplicity, community connection, and local Icelandic heritage.

### Brand Values
- **Local First:** Celebrating small-town identity
- **Connected:** Bringing communities together
- **Authentic:** Real, trustworthy information
- **Accessible:** Easy for everyone to use
- **Nordic:** Clean, minimalist, functional design

## Logo Usage Rules

### Clear Space
Maintain clear space around the logo equal to the height of the church spire (approximately 1/3 of icon height).

```
┌─────────────────────┐
│                     │
│   ┌───────────┐     │
│   │  [LOGO]   │     │ ← Minimum clear space
│   └───────────┘     │
│                     │
└─────────────────────┘
```

### Minimum Sizes
- **Digital:** 32x32px (icon), 120px width (full logo)
- **Print:** 0.5 inch height (icon), 1.5 inches width (full logo)

### Do's
✅ Use the provided SVG files
✅ Maintain aspect ratio
✅ Use on solid backgrounds
✅ Use appropriate color variant for background
✅ Keep the logo legible at all sizes

### Don'ts
❌ Don't distort or stretch the logo
❌ Don't change the colors
❌ Don't add effects (shadows, outlines, glows)
❌ Don't rotate the logo
❌ Don't place on busy backgrounds
❌ Don't recreate or redraw the logo

### Color Usage Guidelines

**Primary Actions**
- Use Blue 500 (`#3b82f6`) for primary buttons, links, and interactive elements
- Hover state: Blue 600 or Blue 400
- Focus state: Add blue ring with opacity

**Status Indicators**
- Success: Green 500 with Green 100 background
- Warning: Amber 500 with Amber 100 background
- Error: Red 500 with Red 100 background
- Info: Blue 500 with Blue 100 background

**Text Hierarchy**
- Headlines: Slate 900 (`#0f172a`)
- Body text: Slate 700 (`#334155`)
- Secondary text: Slate 600 (`#475569`)
- Muted text: Slate 500 (`#64748b`)

**Backgrounds**
- Primary: White (`#ffffff`)
- Subtle: Slate 50 (`#f8fafc`)
- Cards: White with slate borders
- Dark mode: Slate 900 with Slate 800 surfaces

## Typography

### Recommended Fonts

**Primary Typeface:** System UI Stack
```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI',
             'Roboto', 'Helvetica Neue', Arial, sans-serif;
```

**Why System Fonts?**
- Native feel on each platform
- Fast loading (already installed)
- Excellent readability
- Consistent with OS design language

### Type Scale

| Style | Size | Weight | Usage |
|-------|------|--------|-------|
| Display | 56px | 700 | Hero headlines |
| H1 | 40px | 700 | Page titles |
| H2 | 32px | 600 | Section titles |
| H3 | 24px | 600 | Subsection titles |
| H4 | 20px | 600 | Card titles |
| Body Large | 18px | 400 | Important body text |
| Body | 16px | 400 | Default body text |
| Body Small | 14px | 400 | Secondary text |
| Caption | 12px | 400 | Labels, metadata |
| Tiny | 10px | 500 | Uppercase labels |

### Typography Guidelines

**Line Height**
- Headlines: 1.2
- Body text: 1.5
- Captions: 1.4

**Letter Spacing**
- Headlines: -0.02em (tight)
- Body: 0 (normal)
- All caps: 0.05em (loose)

## Brand Voice

### Tone
- **Friendly:** Welcoming and warm
- **Helpful:** Clear and informative
- **Local:** Community-focused
- **Reliable:** Trustworthy and accurate

### Writing Style
- Use second person ("you", "your")
- Keep it concise and scannable
- Use active voice
- Avoid jargon
- Be specific about actions

### Examples

**Good:**
- "Your town's events, all in one place"
- "Never miss what's happening in [Town Name]"
- "Connect with your community"

**Avoid:**
- "Leveraging cutting-edge technology to facilitate municipal engagement"
- "TownApp provides comprehensive solutions for community interaction"

## Export Guidelines

### Converting SVG to PNG

**Using Command Line (ImageMagick):**
```bash
# iOS App Icon
magick -background none -density 300 ios-app-icon-1024.svg -resize 1024x1024 icon.png

# Android Adaptive Icons
magick -background none -density 300 android-adaptive-foreground-432.svg -resize 432x432 adaptive-icon.png
magick -background none -density 300 android-adaptive-background-432.svg -resize 432x432 adaptive-icon-bg.png

# Favicon
magick -background none -density 300 townapp-icon.svg -resize 32x32 favicon-32.png
magick -background none -density 300 townapp-icon.svg -resize 16x16 favicon-16.png
```

**Using Figma/Sketch:**
1. Import SVG file
2. Select artboard
3. Export as PNG at appropriate size
4. For iOS: Ensure no transparency
5. For Android foreground: Keep transparency

**Online Tools:**
- [Inkscape](https://inkscape.org/) - Free, powerful SVG editor
- [Squoosh](https://squoosh.app/) - Image compression
- [SVGOMG](https://jakearchibald.github.io/svgomg/) - SVG optimization

## Accessibility Standards

### WCAG Compliance
All color combinations used in the UI must meet **WCAG AA** standards (4.5:1 for normal text, 3:1 for large text).

### Color Contrast Ratios

**Text on White Background:**
- Slate 900 (`#0f172a`): 16.27:1 ✅ AAA
- Slate 700 (`#334155`): 9.39:1 ✅ AAA
- Slate 600 (`#475569`): 6.79:1 ✅ AAA
- Blue 500 (`#3b82f6`): 9.15:1 ✅ AAA (large text)

**White Text on Color Background:**
- Blue 500 (`#3b82f6`): 2.29:1 ❌ Fails (use darker blue)
- Blue 800 (`#1e40af`): 7.31:1 ✅ AAA
- Indigo 500 (`#6366f1`): 2.52:1 ❌ Fails (use darker indigo)

### Best Practices
- Never use color alone to convey information
- Include text labels with icons
- Provide sufficient contrast for all UI elements
- Test with colorblindness simulators

## Platform-Specific Guidelines

### Web (Next.js)
- Use SVG logos for sharp rendering at all sizes
- Implement dark mode logo switching
- Optimize images with Next.js Image component
- Use CSS custom properties for colors

### Mobile (React Native / Expo)
- Use PNG icons (React Native doesn't support SVG natively)
- Provide @2x and @3x versions for iOS
- Use adaptive icons for Android (separate foreground/background)
- Implement splash screens with logo
